WebGL Data

Indices

  • Numerical values that label vertices.
  • Used to identify the vertices.
  • They are used to draw meshes in WebGL.

Arrays

  • There are no predefined methods in WebGL to render vertices directly.
  • We store vertices manually in JavaScript arrays.
  • We pass them to WebGL rendering pipeline using index buffer.

Buffers

  • Memory areas of WebGL that hold the data.
  • There are various buffers.
  • Vertex Buffer and Index Buffer: used to describe and process the geometry.

Vertex Buffer

  • Stores data about the vertices.

Index Buffer

  • Stores data about the indices.

Frame Buffer

  • Portion of graphics memory that hold the scene data.

  • Contains:

    • width & height of the surface in pixels
    • color of each pixel
    • depth buffer
    • stencil buffer

Mesh

Used to draw 2D or 3D objects

Types: (mode)

  • Points
  • Lines
  • Triangles

Shaders

WebGL uses Embedded System Shader Language (ESSL) that runs on GPU.

We use triangles to construct meshes

The information about these triangles should be transferred from CPU to GPU.

It is a snippet that implements algorithms to get the pixels of a mesh.

  • Shaders are programs for GPU.
  • We define in the shaders how the vertices, transforms, materials, lights and camera interact with one aonther.

Vertex Shader

  • Program called on every vertex.
  • Used to move / transform the geometry.
  • Handles vertext data: coordinates, normals, colors and textures.

Tasks:

  • Vertex transformation.
  • Normal transformation and normalization.
  • Texture coordinate generation.
  • Texture coordinate transformation.
  • Lighting.
  • Color material application.

Fragment Shader

  • A Fragment is the surface of each mesh triangle.
  • Fragment Shader is the code that runs on every pixel on each fragment.
  • It is written to calculate nd fill the color on individual pixels.

Tasks:

  • Operations on interpolated values.
  • Texture access.
  • Texture application.
  • Fog.
  • Color sum.

ES SL Variables

Attributes

  • Hold the input values of the vertex shader program.
  • They point to the vertex buffer object that contains per-vertex data.
    • Each time the vertex shader is invoked, the values of these attributes vary.

Uniforms

  • These are variables that hold the input data that is common for both vertext and fragment shaders: light position, texture, coordinates and color.

Varyings

  • Variables that are used to pass the data from the vertex shader to the fragment shader.
